# Mnemic Code of Conduct

Mnemic is built for people working on coding agents, memory systems, MCP tooling, and related developer infrastructure. The project should stay useful, direct, and safe for contributors with different levels of experience.

## Expected Behavior

- Be respectful and specific when discussing ideas, bugs, benchmarks, and tradeoffs.
- Critique code, evidence, docs, and claims rather than people.
- Assume public issues, pull requests, screenshots, benchmark fixtures, and memory exports may be read by future users.
- Keep security reports, leaked secrets, customer data, private prompts, and production memory exports out of public threads.
- Use reproducible commands, logs, fixtures, and source references when making technical claims.

## Unacceptable Behavior

- Harassment, personal attacks, threats, or sustained disruption.
- Publishing private information, credentials, customer data, or private memory records.
- Misrepresenting benchmark results, security state, package availability, or hosted-service availability.
- Repeatedly ignoring maintainer direction on scope, safety, or evidence requirements.

## Enforcement

Maintainers may edit or hide comments, close issues or pull requests, restrict participation, or report abuse to GitHub when needed. Security-sensitive reports should follow [SECURITY.md](SECURITY.md) rather than public issue threads.

## Scope

This code of conduct applies to project repositories, issues, pull requests, discussions, release artifacts, examples, benchmark fixtures, and community chat spaces controlled by the project.
